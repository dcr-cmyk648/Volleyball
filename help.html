Help

Use this order every game day. It keeps local games from getting overwritten and gives the administrator the file they need.

Game Day Workflow

Start in Stats and make sure the database is current.

If the server button says new games are available, sync before entering anything for today. If it says syncing would delete local games, stop and export the database first.

Go to Play and build the list for today.

Register any new player, then check the boxes next to the players who are here. Pick league level and court type if they apply.

Assign teams.

Use Assign Teams after the present players are checked. If needed, tap a player's name to move them between team colors manually. For multi-team assignments, the intended matchup per the algorithm is Red vs Blue, Green vs Yellow, and so on.

Keep assigning and recording until the session is done.

Scores are not required, but they help make the model more accurate. Enter them if you have them, tap the winning team, and confirm the save. Then assign the next teams and record the next game. Keep repeating that loop until everyone is finished playing.

Export the database and send it to the administrator.

Open Stats, use Export Database, and send the downloaded JSON file so it can be added to the global database.

Do Not Sync After Recording Games

Once game data is entered, do not sync from the server again. Syncing replaces the local database with the server database and can delete games you just recorded.
The safe end-of-night move is Export Database. The safe start-of-night move is sync first, before recording games.
